MXX: Providing touch access for custom controls

Note: I will revise this content to fit the M000 template. If you create a page for "Providing touch access for custom controls" I will then move content there.

Applicability

This technique applies to touch and pointer operating modes.

This technique relates to:

  • Success Criterion 2.5.1 Touch: All functions available by touch are still available by touch after system assistive technology that remaps touch gestures is turned on. (Level A)

Description

Custom controls web interfaces may not be suitable for touch access when the built-in screen reader of mobile operating systems is enabled.

Where default touch gestures do not work because of screen reader gestures that take precedence, ensure that there are equivalent alternative ways of activating or changing the state or value of the control.

Examples

Example 1

In the default operating mode (screen reader not enabled), a slider can be used to change a value by moving the thumb of the slider. When a screen reader is turned on, the direct manipulation does not work. Theerefore a different method is provided to set the value of the slider. Discrete marks along the slider representing a scale of values may be selected by first moving the focus to the mark and then selecting the focued mark by double-tapping.

Tests

Procedure

  1. For each custom control that is operated with touch, check that touch operation is also possible when the screen reader is activated
  2. If touch operation is not possible when the screen reader is activated, check that an equivalent alternative means to carry out the underlying function is provided

Expected Results

  • Check #1 or #2 above are true.
