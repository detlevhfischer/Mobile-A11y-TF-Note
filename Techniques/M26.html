Providing touch access for custom controls

Applicability

This technique relates to touch and pointer operating modes.

This technique relates to:

  • Success Criterion 2.5.1 Touch: All functions available by touch are still available by touch after system assistive technology that remaps touch gestures is turned on. (Level A)

Description

Custom controls web interfaces may not be suitable for touch access when the built-in screen reader of mobile operating systems is enabled.

Tests

Procedure

  1. For each custom control that is operated with touch, check that touch operation is also possible when the screen reader is activated
  2. If touch operation is not possible when the screen reader is activated, check that an equivalent alternative means to carry out the underlying function is provided

Expected Results

  • Check #1 or #2 above are true.
